## 一、🗺️ 更新後的學習路徑圖 (Roadmap 2.0)

為了避免計畫爆炸，採用「揉合」策略，讓不同課程與專案緊密結合：

| 階段 | 主題 | 課程 / 資源 | 學習目標 | 實作項目 | 預計週數 |
|---|---|---|---|---|---|
| **基礎期** | Transformer & Token | HF LLM Course（前三章）| 理解模型架構、token 化、基本微調 | 微調 BERT / DistilBERT | 3~4週 |
| **進階期（RAG）** | 檢索增強生成 | LangChain RAG 實作 | 學會整合 Vector DB、embedding、LLM | RAG 應用 + 知識庫 | 2~3週 |
| **Tool Calling 期** | 函數調用機制 | OpenAI Cookbook + LLM Protocol | 工具定義、LLM 自動調用工具、function calling 標準 | 多工具系統 | 2~3週 |
| **應用期（Agents）** | Agent 邏輯 | HF Agents Course | 基於工具的多步推理、決策流程 | 簡單 Agent（整合工具） | 2~3週 |
| **工程期（Workflows）** | 流程控制 | LangGraph | 學會用狀態圖管理複雜 Agent、避免無限迴圈 | LangGraph Agent 應用 | 2~3週 |
| **部署期** | 企業級應用 | Google Skills 183 + HF MCP | 標準化部署、API 設計、模型服務化 | 完整應用上線 | 2~3週 |

> 核心原則：「揉合」而非「堆砌」——邊學邊做，每個階段都產出實作成果，而非課程走完再找專案。

## 一（原版）、固定學習順序（按照五階段路徑走）

### 基礎期（第 1~4 週）
1. Hugging Face LLM Course（前三章）：Transformer、使用 Transformers、微調
2. 動手學大模型：相關章節強化
3. **專案**：微調一個預訓練模型（BERT / DistilBERT 在自有數據集上）

### 進階期（第 5~7 週）
1. LangChain RAG 實作：Vector DB、embedding、檢索流程
2. **專案**：實現一個完整 RAG 系統（支持上傳文件、提問、回答）

### Tool Calling 期（第 8~10 週）
1. OpenAI Function Calling Protocol：工具定義、LLM 自動選擇工具
2. LangChain / LLM Tool Wrapper：標準化工具調用機制
3. **專案**：實現多工具系統（天氣查詢 + 計算機 + 翻譯 + 自定義工具，LLM 自動選擇）

### 應用期（第 11~13 週）
1. Hugging Face Agents Course：基於工具的 Agent 框架
2. 複雜決策流：多步推理、條件分支、錯誤恢復
3. **專案**：實現簡單 Agent（整合多個工具，執行複雜多步任務）

### 工程期（第 14~16 週）
1. LangGraph：狀態管理、迴圈控制、複雜工作流設計
2. **專案**：用 LangGraph 重構之前的 Agent（更穩定、可控、可視化）

### 部署期（第 17~19 週）
1. Google Skills Path 183：生成式 AI 開發最佳實踐
2. Hugging Face MCP：模型服務標準化
3. **專案**：將完整應用部署上線（API、文檔、監控）

## 二、只分兩件事（以天為單位）

### A. 學習
目標：按順序把核心知識和工具能力補齊。

學習日（140 到 170 分鐘）建議：
1. 20 分鐘：複習昨天重點。
2. 90 到 120 分鐘：主課程學習與實作。
3. 30 分鐘：寫學習紀錄（md）。

學習完成標準（每個階段都用同一套）：
1. 至少完成 1 份重點筆記。
2. 至少完成 1 次可重現實作。
3. 至少整理 1 組常見錯誤與解法。

### B. 做專案
目標：持續把學到的內容轉成可展示成果。

專案日（140 到 170 分鐘）建議：
1. 20 分鐘：確認今天只做一件關鍵任務。
2. 90 到 120 分鐘：開發或調參。
3. 30 分鐘：更新 README/截圖/結果紀錄。

專案日可安排技術補充（視任務需要）：
1. n8n 工作流實作。
2. WebSocket 串接與除錯。
3. 其他專案所需技術（只限與當天專案任務直接相關）。

專案優先順序：
1. 先確保流程可重現（環境、訓練、推論）。
2. 再做效果優化（參數、資料、速度）。
3. 最後做展示包裝（README、成果圖、心得）。

## 四、每週檢查（10 分鐘）

每個階段的檢查重點：

### 基礎期檢查
- [ ] 是否按順序完成 HF LLM Course（前三章）
- [ ] 微調專案是否至少跑通一個 epoch，有 loss 曲線記錄
- [ ] 是否寫了筆記，涵蓋 Transformer、Token、微調三個主題

### 進階期檢查
- [ ] LangChain RAG 流程是否完整跑通（檢索 + 生成）
- [ ] Vector DB 是否正確建立，檢索結果是否合理
- [ ] 專案 README 是否有使用說明、示例、結果展示

### Tool Calling 期檢查
- [ ] 是否理解 function calling protocol（OpenAI、Anthropic 等）
- [ ] 是否能正確定義工具 schema（參數、描述、返回值）
- [ ] LLM 是否能自動判斷何時調用哪個工具
- [ ] 是否能處理工具調用參數缺失或錯誤的情況
- [ ] 多工具系統是否能正確串聯（工具 A 的輸出 → 工具 B 的輸入）

### 應用期檢查
- [ ] HF Agents 代碼是否能完整執行，Agent 邏輯是否清楚
- [ ] Agent 是否能基於工具庫正確決策（相比 Tool Calling 期增加決策複雜度）
- [ ] 是否有複雜多步推理的實際案例驗證

### 工程期檢查
- [ ] LangGraph 狀態圖是否清晰、邏輯是否完整
- [ ] 是否實現了控制流（條件分支、迴圈退出條件）
- [ ] 相比前版本，是否更穩定、更好除錯

### 部署期檢查
- [ ] API 是否定義清楚，文檔是否完整
- [ ] 是否有監控、日誌、錯誤處理
- [ ] 應用是否能在外部環境跑起來（非開發環境）
